# v2.4.5 — Stabilization / Verification

**Tag:** `v2.4.5` (merge `6b517fa`) · **Status:** Released with carry-forward verification debt (production-readiness gate, not a public release)

## What actually shipped

The architecture-foundation and honesty release. Every AI answer is made honest before any new capability is added:

- **Provider lifecycle + Plaid sync/reconcile**, admin diagnostics, expand-history workflow, SAL dual-write.
- **BALANCE_ONLY leak closed on both surfaces** — AI context (KD-1) and UI transaction reads (KD-15) now honor `VisibilityLevel` through the single canonical predicate `lib/ai/visibility.ts`.
- **Live LLM output validation** (KD-2) — the shadow validator promoted to live enforcement (`shadow | annotate | block`, default `annotate`); a reply quoting a figure absent from context is flagged.
- **Rate limiting** on auth + chat (KD-3).
- **Write atomicity** — `db.$transaction` around multi-table SAL writes (KD-4); HOME partial-unique index under concurrency (KD-5).
- **AI financial-context honesty** — checked category-rollup invariant (KD-17) and the generalized non-attribution guardrail (KD-18): the AI never fabricates a per-account/card breakdown the membership validator cannot catch.
- **Unified test runner + CI** (`npm test` → `scripts/run-tests.ts`).

## Known gaps

- Dynamic-window / follow-up / drilldown characterization suites — **re-homed to AI5-1** (bounds-not-dollars); do not build twice.
- Fallback-hit observability counter — re-homed to later PO1.x observability.
- `.env.example` drift (`AI_OUTPUT_VALIDATION_MODE`, `SECURITY_PRICES_ENABLED`, `INVESTMENT_IMPORTS_ENABLED`, `FLOWTYPE_SHADOW`; `TIINGO_API_KEY` referenced only in comments).

## Migration requirements

Additive migrations only, including the KD-5 HOME partial-unique index (`20260702170000_kd5_home_partial_unique_index`). The index migration **fails loud** if pre-existing duplicate HOME links are present — run `scripts/dedupe-home-links.ts` first. No breaking changes; existing data preserved.

## Production readiness

**Not ready for external users.** The tag was cut and v2.5 opened before the verification debt closed. That debt has since been largely closed or deliberately re-homed — the binding constraint is no longer "all v2.4.5 debt retired" but the current beta-blocker list (consent/disclosure + operational floor + config verification) tracked in the production-readiness audit.
