01 — Foundation

Brand foundation

Fourth Meridian's mark is a globe crossed by meridian lines, fixed by a north star, rendered in brass and ink. Every design decision below — the deep navy surfaces, the brass accent, the precision of the grid — extends that mark rather than decorating around it.

Primary mark — dark surfaces
Inverse mark — light surfaces
Brand personality
Premium Intelligent Modern Calm Trustworthy Wealth Precision Atlas Navigation Meridian
What this system is
A locked set of tokens — color, type, elevation, glass, radius, spacing, motion, icon, chart, semantic, a11y — that every Fourth Meridian surface inherits from.
What it is not
Not a page layout. Not a component library. No dashboards, no nav patterns, no screen flows are specified here — that is v2 work, scoped separately.
Voice alignment
Calm, not urgent. Specific, not numerous. This visual language should read the same way Fourth Meridian's copy sounds — see fourth-meridian-product-language.md.
02 — Foundation

Color system

Six ramps. Atlas Ink is the neutral spine; Brass is the brand accent lifted directly from the mark; Meridian is the primary interactive color. Click any swatch to copy its hex value. Dark is the default surface — Paper is the light-mode base.

Atlas Ink — neutral
Brass — brand accent
Meridian — primary interactive
Emerald — success / gain
Coral — danger / loss
Violet — secondary data category
Paper — light mode base
Usage rules
Meridian is the only color used for primary actions, links, and focus rings — never recolor a primary button with Brass or Violet. Brass is reserved for premium markers, the wordmark, and featured/AI surfaces — never for status or data. Emerald and Coral are reserved exclusively for actual gain/loss and pass/fail states — never reused for categorical chart series. At least 90% of any screen should be Atlas Ink; color is the exception, not the texture.

05 — Foundation

Elevation

On near-black surfaces, drop shadow alone barely reads. Elevation here is communicated by border luminance and inner highlight as much as by shadow depth — five levels, lowest to highest.

E0 — Base
Page background. No shadow, no border.
E1 — Resting
Default card. Hairline border + 1px inset highlight.
E2 — Raised
Hover / interactive card. Soft 28px shadow.
E3 — Overlay
Popovers, menus. 44px shadow, stronger border.
E4 — Modal
Sheets, dialogs. 70px shadow + scrim behind.
06 — Foundation

Glass material system — "Atlas Glass"

Four blur weights, each with a specular top edge to simulate light catching a glass plane (iOS 26/27 liquid-glass inspired). Tint variants wash a 12–18% hue over the base material for status or premium surfaces.

11 — Foundation

Chart colors

A fixed eight-color categorical order — the same series always gets the same color across every view. Sequential and diverging ramps for heatmaps and density views. Red and green are reserved for real gain/loss, never for arbitrary categories.

Categorical — fixed order
Example — allocation by category
Sequential — density / heatmap
Diverging — gain ↔ loss
Gridlines: Ink-700 at 12% opacity. Axis labels: Caption / Ink-300. Maximum six series per chart before splitting into small multiples.
