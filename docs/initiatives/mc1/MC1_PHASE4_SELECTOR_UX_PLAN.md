## 0. Executive summary

Phase 4 is presentation over a finished machine. The one structural finding: `DEFAULT_DISPLAY_CURRENCY` is baked into ~12 components' `Intl.NumberFormat` calls as a **module constant**, so the long-promised "swap the source" is really a small React context ‚Äî a `DisplayCurrencyProvider` fed by the server from `Space.reportingCurrency`, consumed via a `useDisplayCurrency()` hook that replaces the constant one component at a time (USD Spaces render pixel-identically). Everything else is scoped decisions: selectors in the two places users already edit these things (ManageSpaceModal for the Space, Settings‚ÜíProfile for the User default), a quiet "‚âà est." treatment reusing the perspective widget's existing pattern, read-time conversion of off-stamp chart points (no second snapshot flag ‚Äî entry finding #1 resolved: **no schema**), the holdings assembler threaded at last (F-5), a serialized context in the Space-transactions API payload (F-6), and a one-line AI currency label with a single estimation disclosure in the KD-18 pinned-wording style. Nine slices; the ephemeral view override is deliberately last and cuttable.

## 1. Investigation findings

1. **Formatter seam is a constant, not a parameter.** Charts (`CashChart`, `NetWorthChart`, `PortfolioHistoryChart`, ‚Ä¶) and dashboard components inline `currency: DEFAULT_DISPLAY_CURRENCY` in `Intl.NumberFormat`. A runtime per-Space value therefore needs a provider/hook (D-1); the `lib/currency.ts` header's "no component changes needed" promise was written for a build-time swap and is superseded.
2. **Selector homes exist.** `ManageSpaceModal` already edits name/category/isPublic through the Phase 3-extended PATCH (allowlist live, 400s on invalid); `app/(shell)/dashboard/settings/profile` is the natural User-default home (needs its update route extended ‚Äî implementation detail for the slice).
3. **Chart readers are field-selective.** `lib/data/snapshots.ts` selects specific float fields ‚Äî mixed-stamp handling extends these reads with `reportingCurrency` and converts server-side (readers are server code; no client rate access needed).
4. **F-5 confirmed:** `lib/ai/assemblers/holdings.ts` sums `h.value` raw; `Holding.currency` (Phase 0) is not selected. **F-6 confirmed:** `app/api/spaces/[id]/transactions/route.ts` exists ‚Äî the payload-extension target.
5. **Audit precedent:** the Space PATCH already writes `AuditLog` (`SPACE_UPDATE` + metadata) ‚Äî currency changes ride it with a metadata enrichment, no new action type.

## 4. Validation gates (phase-wide, every slice)

1. **USD invariance:** every slice re-verifies a USD Space renders pixel-/byte-identically (provider defaults, empty payloads, silent flags, homogeneous charts).
2. **Non-USD correctness:** a fixture EUR Space shows converted, labeled aggregates; native itemized rows; honest flags on misses.
3. **No mutation:** grep proofs from Phase 3 closeout re-run at every slice (lib/money write-free; snapshot writers' write-shape unchanged).
4. **Suites:** `npm test` (all MC1 gates + kd17/kd18/privacy/validator/perspective), `tsc --noEmit`, `npm run lint` ‚Äî green per slice.
5. **Selector behavior:** updates persist, seed correctly (copy-once), audit correctly, reject invalid codes.

## 5. Rollback strategy

- **Data-level (global):** `UPDATE "Space" SET "reportingCurrency" = 'USD'` ‚Äî every aggregate re-renders in USD on the next read; indicators go silent; charts go homogeneous. Works with zero deploys, unchanged from Phase 3.
- **UI-level:** each slice's surface is independently removable (provider unmount ‚Üí USD constant; hidden selectors leave the API in its Phase 3 state; indicators/labels are leaf components).
- **Partially mixed snapshot history:** needs no rollback ‚Äî stamps are facts, and both renderers (stamp-blind today, stamp-aware Slice 4) are read-only interpretations. Reverting Slice 4 simply returns to stamp-blind rendering; nothing stored has changed.
- **API payload (Slice 6):** additive response field; omitting it restores native panel sums via the optional prop.

## 6. Constraints honored

No Transaction/Holding amount is touched; no write-time normalized columns; no snapshot rewrite (Slice 4 is read-path only, and resolved entry finding #1 as **no new schema**); no Merchant Intelligence; nothing beyond selector/presentation. Every slice independently shippable and revertible.
