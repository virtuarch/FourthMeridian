## 1. Project overview

Fourth Meridian is a personal finance platform built on Next.js 16 / Prisma / PostgreSQL, deployed to Vercel (sin1) with Supabase Postgres.

What exists and works today:

- **Accounts & sync** — Plaid Link, cursor-based incremental transaction sync with retry and manual-refresh cooldown, automatic duplicate-account reconciliation that survives Plaid reissuing account/transaction IDs, crypto wallet tracking by address, CSV import (MVP), soft-delete lifecycle throughout.

- **AI analyst** — Space-scoped chat: deterministic intent routing, dynamic transaction windows with follow-up carry-forward, deterministic monthly rollups / merchant normalization / spending trends, a deterministic financial assessment engine (data quality, cash flow, debt, liquidity, capital allocation, risk/opportunity), knowledge-gap capture, provenance-first prompting. LLM: OpenAI gpt-4o-mini behind a single provider boundary.
- **Security** — AES-256-GCM with HKDF per-purpose key derivation, hashed password-reset tokens, TOTP + recovery codes, session management with revocation cache, append-only audit log, SYSTEM_ADMIN role with a fully separate admin surface and redacted provider diagnostics.

## 11. Executive summary

**Architecture maturity: high for its stage.** The expensive-to-retrofit boundaries — tenancy, encryption, auth chokepoints, the LLM provider seam — are correct and consistently enforced. The AI layer's deterministic-first design (the model narrates pre-computed, provenance-carrying facts; it never calculates) is a genuine differentiator executed above industry norm.

**Launch readiness: materially closer to private beta than the previous STATUS suggested.** The core platform architecture is largely built. As of the 2026-07-15 Closure truth-up, remaining work concentrates as follows: **v2.5 (architecture)** — two genuine items left, legacy `Account` runtime-read retirement + `SpaceDashboard.tsx` host decomposition (the perspective/shell satellites are already decomposed; Atlas adoption is substantially met for product surfaces). **v2.5.5 (Financial Intelligence)** — the canonical aggregation architecture is **substantially landed** (CF-3 `DayFacts` is the canonical projection; Summary/History/Calendar parity is test-enforced; `FLOW_COST` duplication resolved); what remains is **semantic convergence + test enforcement**, not construction. **OPS legal/beta systems** — S9/S10 are substantially built; remaining is **operational closure + consent/disclosure + production configuration verification**. **AI-5** — the deterministic substrate is ready for early zero-schema foundation work, while **persistent conversational state is the major unbuilt AI layer** (no `conversationId` exists; the eight-failure corpus is uncommitted). **The binding gap is now verification / configuration / operations more than large feature construction** — but Fourth Meridian is **not yet ready for external users**, and public-production readiness is not overstated: the beta-blocker list in §6 (consent, LLM disclosure, Sentry, uptime, backup drill, invite-only + Turnstile + Plaid production verification) is real and must close first.

**Largest strengths:** deterministic AI assessment pipeline; Plaid failure-mode resilience; migration discipline; security architecture fundamentals.

**Largest weaknesses:** verification/configuration debt (invariants asserted in comments; production config + operational floor unverified); solo bus factor; the unbuilt persistent-conversation AI layer. *(The former "no operational substrate for scheduled intelligence" weakness is retired — OPS-4 shipped the typed job platform, registry, dispatcher, JobRun ledger, and dead-job detection; scheduled AI advice generation remains a v2.6b feature, not a missing substrate.)*

**Biggest risks:** documentation reverting to fragmentation if this file is not maintained; residual false-positive tuning on the now-live validator before `block` can be trusted as default. (Items previously headlining this list are now closed — the BALANCE_ONLY leak via KD-1/KD-15, the non-atomic SAL writes via KD-4, HOME uniqueness under concurrency via KD-5, and *live* output validation via KD-2 — all fixed and committed. The prior "ambient features speaking unprompted before live validation exists" risk is retired: the validator is enforcing, and v2.6b ambient behavior still gates on it — and, as of the 2026-07-02 roadmap revision, on the v2.6a conversation layer as well.)

**Recommendation:** v2.4 is closed and `v2.4.5` tagged (`6b517fa`) — the two absorbed additions landed (KD-17 and the KD-18 honesty guardrail — both correctness defects, which is what the gate is for; **both fixed & committed, `3988510`**; the KD-18 *capability* is ratified into v2.5.5 FlowType, not absorbed), with the v2.4.5 verification debt (named test suites, observability counters, max-50 copy) carried forward into the now-open v2.5 window; keep v2.5 as seam-closure plus design foundation; make the data semantically sound in v2.5.5; make conversations coherent in v2.6a; let v2.6b earn ambient behavior behind both the validator and the conversation layer; launch at v3.0 with zero new surface.
